슬라이드 1. 표지 (운전자 졸음 방지 시스템)

안녕하십니까. A팀입니다.
저희는 운전자 상태 모니터링(DSM, Driver State Monitoring) 기반의 졸음 및 하품 감지 시스템을 설계·구현했습니다.
본 프로젝트의 핵심은 세 가지입니다.
첫째, 주·야간을 고려한 정제 데이터 엔지니어링, 둘째, EAR/MAR + CNN 하이브리드 판정, 셋째, RK3588 NPU 최적화(RKNN 변환)를 통한 실시간성 확보입니다.
지금부터 데이터 구축부터 온보드 실증까지 순서대로 설명드리겠습니다.


슬라이드 2. 프로젝트 개요

프로젝트의 출발점은 단순합니다.
졸음운전은 운전자가 외부 자극에 반응하지 못하는 의식 저하 상태에서 발생하며, 제동 반응이 지연되어 사고 파급력이 매우 큽니다.
기존 사후 대응 중심 접근 대신, 저희는 실시간 선제 경보를 목표로 했습니다.

오른쪽 통계에서도 보이듯 졸음운전 사고는 계절·시간대에 관계없이 지속적으로 발생합니다.
따라서 저희는 단순 데모가 아닌, 실제 주행 환경 적용을 전제로 데이터-모델-시스템-예외처리까지 일관된 파이프라인을 구성했습니다.


슬라이드 3. 목차

발표는 다섯 파트로 구성됩니다.
1) 프로젝트 개요, 2) 팀 구성 및 역할, 3) 시스템 수행 절차와 방법, 4) 수행 결과, 5) 자체 평가 의견입니다.
특히 심사위원분들이 관심 있으실 임계값 설계 근거, 하이브리드 판정식, NPU 배포 과정을 중심으로 설명드리겠습니다.


슬라이드 4. 시스템 수행 절차 및 방법(일정표)

개발은 약 3주간 진행했고, 기획-데이터 수집-모델링-온보드 테스트의 4단계로 관리했습니다.
초기에는 모델별 역할을 분리해 병렬로 검증했고, 중반부에는 정확도/지연시간(latency) 트레이드오프를 기준으로 후보를 축소했습니다.
후반부에는 Orange Pi 5 Plus에서 실시간 추론을 수행하며 오탐/미탐 케이스를 반복 튜닝했습니다.
핵심은 모델 성능만이 아니라, 실시간 경보 시스템으로서의 안정성까지 포함해 평가한 점입니다.


슬라이드 5. 구성 및 역할

팀 역할은 기능 단위로 분리했습니다.
저는 전체 시스템 통합과 RKNN 환경 구성, 그리고 MediaPipe/dlib 기반 전처리 파이프라인을 담당했습니다.
선우 팀원은 데이터 수집·정제와 YOLO/CNN 학습 및 성능 검증을 담당했고,
여신구 팀원은 Teachable Machine 기반 비교 실험과 구조 설계, 표준 사례 분석을 수행했습니다.
김동욱 팀원은 부분 데이터 모델 학습과 디바이스 그래프 테스트로 온보드 검증을 지원했습니다.
이렇게 분업해 실험 속도와 재현성을 동시에 확보했습니다.


슬라이드 6. 수행 경과(핵심 코드: 랜드마크/임계값)

여기서는 판정 로직의 기반이 되는 수치화를 설명드리겠습니다.
dlib 68-landmark에서 눈(36~47), 입(48~67) 좌표를 추출하고,
눈은 EAR(Eye Aspect Ratio), 입은 MAR(Mouth Aspect Ratio)로 정규화된 기하학적 특징을 계산합니다.

임계값은 경험치가 아니라 반복 실험으로 고정했습니다.
- EAR 임계값: 0.22
- MAR 임계값: 0.50
- 졸음 지속 경보 시간: 10초
- 운전자 이탈 리셋: 3초
이 설정으로 민감도와 특이도 균형을 맞췄습니다.


슬라이드 7. 수행 경과(데이터셋 -> CNN 학습 -> ONNX/RKNN)

데이터셋은 총 775장으로, Open 393장 / Closed 382장으로 클래스 밸런스를 맞췄습니다.
특히 학습 단계에서 dlib 자동 크롭을 쓰지 않고, 눈 ROI를 224x224로 수동 정제해 데이터 순도를 높였습니다.

모델은 MobileNetV2 기반 전이학습을 적용했습니다.
배치 16, 에포크 50, Dropout 0.6, 밝기 증강 0.5~1.5로 저조도 일반화를 확보했습니다.
이후 Keras(.h5) -> ONNX -> RKNN 2단계 변환으로 RK3588 NPU에 배포 가능 형태로 최적화했습니다.


슬라이드 8. 수행 경과(객체 인식/랜드마크/DSM UI)

실시간 파이프라인은 다음과 같습니다.
프레임 입력 -> 얼굴 추적 -> 랜드마크 추출 -> 눈/입 ROI 생성 -> CNN 추론 및 EAR/MAR 결합 판정.
우측 영상처럼 UI에 FPS, CNN 확률, EAR, MAR를 동시에 표시해 판정 근거의 가시성(explainability)을 확보했습니다.

또한 눈·입 확대 뷰를 제공해 센서 노이즈나 조명 변화 시에도 운영자가 상태를 즉시 확인할 수 있게 했습니다.


슬라이드 9. 수행 경과(운전자 추적/하품 경보/상태 표시)

이 슬라이드는 실제 이벤트 처리 로직입니다.
하품은 MAR > 0.50을 기준으로 누적 카운팅하고, 3회 이상 시 경보를 발생시킵니다.
경보 메시지는 시야 방해를 줄이기 위해 일정 시간 후 자동 소거됩니다.

얼굴 박스는 상태별로 색을 분리했습니다.
정상은 녹색, 위험 상태는 적색으로 즉시 피드백하며,
운전자 이탈 시에는 추적 상태를 유지·복구하는 로직으로 불필요한 경보 진동을 줄였습니다.


슬라이드 10. 수행 경과(졸음 래칭/각성 확인/이탈 처리)

핵심 차별점은 안전 로직의 상태기계화(stateful logic)입니다.
단순히 눈을 감았다고 바로 끝내지 않고,
- 10초 이상 연속 감김 시 강경 경보
- 인식이 잠시 끊겨도 직전이 졸음 상태면 경보 유지(Latching)
- 경보 해제는 정상 상태가 10초 연속 유지될 때만 허용(Wake-up confirmation)
으로 설계했습니다.

즉, 순간적인 눈 뜸이나 포즈 변화에 흔들리지 않는 히스테리시스 기반 안정 제어를 구현했습니다.


슬라이드 11. 수행 경과(Orange Pi 5 Plus 성능)

온보드 테스트는 Orange Pi 5 Plus, 즉 RK3588 NPU 기반으로 수행했습니다.
CPU-only 대비 NPU 추론에서 프레임 처리율이 크게 향상되어 실시간 운용이 가능했습니다.
또한 RKNN 변환 후 모델 크기는 약 15MB에서 5.8MB 수준으로 감소해,
임베디드 환경에서 메모리 압박과 배포 부담을 동시에 줄였습니다.

정리하면, 정확도 유지와 실시간성이라는 상충 조건을 하드웨어-소프트웨어 코디자인으로 맞춘 결과입니다.


슬라이드 12. DSM 시연 영상

지금 보시는 영상은 통합 시스템 실시간 동작 화면입니다.
눈 감김, 하품, 이탈 상황에서 각각 경보 로직이 정상적으로 트리거되고,
정상 복귀 시에는 안정화 타이머를 거쳐 경보가 해제됩니다.
즉, 단순 인식 데모가 아니라 상태 전이와 예외처리까지 포함한 운영 가능한 프로토타입임을 확인하실 수 있습니다.


슬라이드 13. 자체 평가 의견(한계 및 개선)

현재 한계는 네 가지로 정리했습니다.
첫째, 측면 얼굴에서 랜드마크 손실, 둘째, 조명 간섭 및 오인식, 셋째, 마스크/선글라스 환경 취약성, 넷째, 장시간 구동 시 발열입니다.

개선 방향은 다음과 같습니다.
- 측면 데이터와 head-pose 보강으로 강건성 향상
- IR 센서 연동 및 조도 보정으로 저조도 성능 개선
- 부분 특징 기반 멀티모달 확장으로 가림 대응
- 모델 경량화 + 쿨링 최적화로 장시간 안정 운용
궁극적으로는 차량 환경에서의 신뢰도 높은 실시간 DSM으로 고도화하겠습니다.


마무리 멘트 (질의응답 직전)

정리하면 저희 시스템은
1) 정제된 주·야간 밸런스 데이터셋,
2) EAR/MAR/CNN 융합 하이브리드 판정,
3) RKNN 기반 온디바이스 실시간 추론,
4) 래칭·안정화·리셋을 포함한 안전 중심 예외처리를 통해,
실제 운행 환경을 고려한 졸음 감지 프레임워크를 제시했습니다.
이상 발표를 마치겠습니다. 감사합니다.
